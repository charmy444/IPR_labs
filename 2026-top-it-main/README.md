# Курс: Инструменты промышленной разработки

Добро пожаловать в репозиторий курса "Инструменты промышленной разработки"!

## Описание курса

Этот курс посвящен изучению современных инструментов и практик, используемых в промышленной разработке программного обеспечения. В рамках курса рассматриваются системы контроля версий, методологии совместной работы, контейнеризация приложений, оркестрация контейнеров и другие важные аспекты профессиональной разработки.

## Лабораторная работа №1: Расширенные возможности ветвления в Git

Эта лабораторная работа посвящена расширенным возможностям ветвления в Git. В рамках работы рассматриваются:
- Продвинутые стратегии ветвления
- Работа с merge и rebase
- Управление историей коммитов
- Решение конфликтов слияния
- Работа с удаленными репозиториями

Подробная документация и задания находятся в [lab1/README.md](lab1/README.md).

## Лабораторная работа №2: Docker - Контейнеризация приложений

В этой лабораторной работе изучаются основы контейнеризации приложений с использованием Docker:
- Основные команды Docker
- Создание Dockerfile
- Работа с образами и контейнерами
- Оптимизация образов
- Публикация образов в Docker Hub
- Использование bind mounts для разработки

Подробная документация и задания находятся в [lab2/README.md](lab2/README.md).

## Лабораторная работа №3: Docker Compose и Multi-stage Builds

Третья лабораторная работа посвящена созданию многоконтейнерных приложений и оптимизации Docker образов:
- Введение в Docker Compose
- Работа с несколькими сервисами
- Сети и volumes в Docker Compose
- Multi-stage builds для оптимизации образов
- Использование профилей и расширенных возможностей

Подробная документация и задания находятся в [lab3/README.md](lab3/README.md).

## Лабораторная работа №4: Непрерывная интеграция и доставка (CI/CD) в GitLab

Четвертая лабораторная работа посвящена освоению принципов непрерывной интеграции и доставки с использованием GitLab CI/CD:
- Основы CI/CD и архитектура GitLab CI/CD
- Конфигурация .gitlab-ci.yml и настройка GitLab Runner
- Создание пайплайнов с unit-тестами для различных языков программирования
- Сборка и публикация Docker-образов в Container Registry
- Работа с артефактами, кэшем и оптимизация пайплайнов
- Переменные окружения, секреты и управление окружениями
- Триггеры, расписание и продвинутые техники CI/CD
- Multi-project pipelines, security scanning и blue-green deployment

Подробная документация и задания находятся в [lab4/README.md](lab4/README.md).

## Лабораторная работа №5: Основы Kubernetes

Пятая лабораторная работа посвящена оркестрации контейнеров в Kubernetes. В рамках работы рассматриваются:

- Локальный кластер (Docker Desktop Kubernetes и аналоги), контекст `kubectl`, базовая диагностика
- Основные объекты API: Pods, Deployments, ReplicaSet, Services, Namespaces, StatefulSets, Ingress
- Манифесты YAML, ConfigMap и Secret, пробы и стратегии обновления Deployment
- Связь с лабораторной №4: образ из Container Registry, `imagePullSecrets`, перенос приложения в кластер
- Эталонный пример [telegram-support-bot](lab5/examples/telegram-support-bot/README.md); углубление по Kustomize/Helm — в лабораторной №6

Подробная документация и задания находятся в [lab5/README.md](lab5/README.md).

## Лабораторная работа №6: Kustomize и Helm

Шестая лабораторная работа закрепляет декларативное описание приложений и разделение **приложения** и **инфраструктуры**. В рамках работы рассматриваются:

- **Kustomize**: база манифестов и overlays (например dev/prod) без шаблонного языка в YAML
- **Helm**: chart, шаблоны, несколько values-файлов для окружений
- Вынесение состоятельной инфраструктуры (PostgreSQL) в отдельный каталог: StatefulSet, Headless Service, PVC, контракт подключения для приложения
- Согласование с лабораторной №5: манифесты приложения в примере бота, инфраструктура — в [telegram-support-infra](lab6/examples/telegram-support-infra/README.md)

Подробная документация и задания находятся в [lab6/README.md](lab6/README.md).

## Лабораторная работа №7: Observability

Седьмая лабораторная работа посвящена **наблюдаемости** приложений и платформы. В рамках работы рассматриваются:

- Метрики в формате **Prometheus**, модель метрик и скрейпинг
- **Grafana**: источники данных, дашборды, запросы PromQL
- Распределённый трейсинг: **OpenTelemetry**, OTLP, приём трейсов в **Grafana Tempo**
- Локально: Docker Compose — приложение со стеком в [telegram-support-bot](lab5/examples/telegram-support-bot/README.md) или только стек в [lab7/examples/telegram-support-observability](lab7/examples/telegram-support-observability/README.md)
- В Kubernetes: **kube-prometheus-stack**, `ServiceMonitor`, развёртывание Tempo (см. тот же каталог примера)

Подробная документация и задания находятся в [lab7/README.md](lab7/README.md).
