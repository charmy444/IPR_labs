# Настройка SSH-подключения к GitLab MAI

## Зачем нужно SSH-подключение?

SSH (Secure Shell) — это протокол сетевой безопасности, который обеспечивает безопасное подключение к удаленным серверам. При использовании SSH-подключения к GitLab MAI ваши данные передаются в зашифрованном виде, что значительно повышает безопасность по сравнению с HTTPS.

## Генерация SSH-ключа

### Шаг 1: Проверка наличия существующих ключей

Перед генерацией новых ключей проверьте, нет ли уже существующих SSH-ключей:

```bash
ls -al ~/.ssh
```

Если вы видите файлы с именами `id_rsa`, `id_ed25519`, `id_ecdsa` и т.д., возможно, у вас уже есть ключи.

### Шаг 2: Генерация нового SSH-ключа

Для генерации нового SSH-ключа используйте команду `ssh-keygen`. Мы рекомендуем использовать алгоритм Ed25519, который обеспечивает хорошую безопасность и производительность:

```bash
ssh-keygen -t ed25519 -C "your.email@mai.ru"
```

При выполнении команды система запросит:

1. **Место сохранения ключа**: По умолчанию ключи сохраняются в `~/.ssh/`. Можно оставить путь по умолчанию, нажав Enter.
2. **Парольную фразу**: Рекомендуется установить парольную фразу для дополнительной безопасности. Если не хотите использовать парольную фразу, просто нажмите Enter.

### Шаг 3: Запуск ssh-agent

Запустите ssh-agent в фоновом режиме:

```bash
eval "$(ssh-agent -s)"
```

### Шаг 4: Добавление ключа в ssh-agent

Добавьте ваш SSH-ключ в ssh-agent:

```bash
ssh-add ~/.ssh/id_ed25519
```

Если вы использовали другое имя файла для ключа, замените `id_ed25519` на соответствующее имя.

## Добавление публичного ключа в GitLab MAI

### Шаг 1: Копирование публичного ключа

Скопируйте содержимое публичного ключа в буфер обмена:

```bash
cat ~/.ssh/id_ed25519.pub
```

### Шаг 2: Добавление ключа в профиль GitLab

1. Перейдите на [gitlab.mai.ru](https://gitlab.mai.ru)
2. В правом верхнем углу нажмите на свой аватар и выберите "Edit profile"
3. В левом меню выберите "SSH Keys"
4. В поле "Key" вставьте содержимое публичного ключа
5. В поле "Title" введите понятное название ключа (например, "Ноутбук MacBook Pro")
6. При необходимости установите дату истечения срока действия ключа
7. Нажмите "Add key"

## Проверка SSH-подключения

После добавления SSH-ключа в GitLab MAI проверьте подключение:

```bash
ssh -T git@gitlab.mai.ru -з 2222
```

Если подключение установлено успешно, вы получите сообщение:

```
Welcome to GitLab, @username!
```

## Устранение неполадок

### Проблемы с правами доступа к файлам ключей

SSH может отклонить использование ключа, если права доступа к файлам неправильные. Установите правильные права:

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

### Использование нестандартного порта

Если вам нужно использовать нестандартный порт для SSH-подключения, добавьте следующую запись в файл `~/.ssh/config`:

```
Host gitlab.mai.ru
  Port 22
  User git
```

### Множественные SSH-ключи

Если у вас несколько SSH-ключей для разных сервисов, вы можете настроить их в файле `~/.ssh/config`:

```
Host gitlab.mai.ru
  IdentityFile ~/.ssh/id_ed25519_mai
  User git

Host github.com
  IdentityFile ~/.ssh/id_ed25519_github
  User git
```

## Настройка SSH для пользователей Windows

### Способ 1: Использование PuTTY

1. Скачайте и установите PuTTY с официального сайта: https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html
2. Запустите PuTTYgen
3. Выберите тип ключа: RSA, 2048 бит
4. Нажмите "Generate" и следуйте инструкциям
5. Сохраните приватный ключ (с расширением .ppk)
6. Скопируйте публичный ключ из верхнего поля
7. Добавьте публичный ключ в GitLab MAI по инструкции выше
8. В PuTTY настройте подключение к gitlab.mai.ru, указав приватный ключ в разделе Connection → SSH → Auth

### Способ 2: Использование Git Bash

1. Откройте Git Bash
2. Выполните команду: `ssh-keygen -t rsa -b 2048 -C "your.email@mai.ru"`
3. Следуйте инструкциям, оставив путь по умолчанию
4. Укажите passphrase для дополнительной безопасности
5. Запустите ssh-agent: `eval $(ssh-agent -s)`
6. Добавьте ключ в агент: `ssh-add ~/.ssh/id_rsa`
7. Скопируйте публичный ключ: `cat ~/.ssh/id_rsa.pub`
8. Добавьте публичный ключ в GitLab MAI по инструкции выше
