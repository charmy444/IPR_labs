# Авторизация через Access Token

## Что такое Personal Access Token?

Personal Access Token (PAT) — это альтернатива паролю для аутентификации в GitLab. Токены обеспечивают более гибкий контроль доступа и повышают безопасность по сравнению с использованием паролей.

## Преимущества использования токенов

- **Гибкость прав доступа**: Можно выдать токену только необходимые права
- **Ограниченный срок действия**: Токены можно настроить на автоматическое истечение
- **Легкий отзыв доступа**: При необходимости токен можно отозвать в любой момент
- **Аудит безопасности**: Легко отслеживать, какие токены активны и где используются

## Создание Personal Access Token

### Шаг 1: Доступ к настройкам токенов

1. Войдите в свой аккаунт на [gitlab.mai.ru](https://gitlab.mai.ru)
2. В правом верхнем углу нажмите на аватар и выберите "Settings"
3. В левом меню выберите "Access Tokens"

### Шаг 2: Настройка токена

1. Введите имя токена (например, "Git CLI Access" или "CI/CD Pipeline")
2. Установите срок действия:
   - Для постоянного использования: 90 дней
   - Для временных задач: 7-30 дней
   - Не рекомендуется устанавливать "без срока действия"
3. Выберите необходимые права (scopes):

| Право | Описание |
|-------|---------|
| `read_repository` | Чтение репозитория (клонирование, получение изменений) |
| `write_repository` | Запись в репозиторий (отправка изменений, создание веток) |
| `read_registry` | Чтение контейнеров из Container Registry |
| `write_registry` | Запись контейнеров в Container Registry |
| `api` | Доступ к API GitLab |

4. Нажмите "Create personal access token"

### Шаг 3: Сохранение токена

1. Скопируйте сгенерированный токен (он покажется только один раз!)
2. Сохраните токен в безопасном месте
3. Никогда не коммитьте токен в репозиторий

## Использование Access Token

### При клонировании репозитория

```bash
git clone https://oauth2:<your_access_token>@gitlab.mai.ru/username/project-name.git
```

### При настройке удаленного репозитория

```bash
git remote set-url origin https://oauth2:<your_access_token>@gitlab.mai.ru/username/project-name.git
```

### Через кеш учетных данных

Для того чтобы не вводить токен каждый раз, настройте кеш учетных данных:

```bash
# Для хранения в памяти на 1 час (рекомендуется)
git config --global credential.helper cache

# Для постоянного хранения (менее безопасно)
git config --global credential.helper store
```

После настройки кеша, при первом обращении к репозиторию введите:
- Username: `oauth2`
- Password: `<ваш_access_token>`

## Безопасность Access Token

### Рекомендации по безопасности

- **Минимальные привилегии**: Выдавайте токену только необходимые права
- **Ограниченный срок**: Устанавливайте разумный срок действия
- **Регулярное обновление**: Меняйте токены каждые 30-90 дней
- **Немедленный отзыв**: Отзывайте токены при смене устройства или подозрении на компрометацию
- **Не передавайте токены**: Никогда не отправляйте токены по небезопасным каналам

### Обнаружение утечек

Если вы случайно закоммитили токен:

1. Немедленно отзовите токен в настройках GitLab
2. Измените все данные, к которым мог получить доступ токен
3. Обновите историю Git, удалив коммит с токеном:
   ```bash
   git filter-branch --force --index-filter \
   'git rm --cached --ignore-unmatch path/to/file/with/token' \
   --prune-empty --tag-name-filter cat -- --all
   ```
4. Уведомите администратора системы при необходимости

## Сравнение методов аутентификации

| Метод | Безопасность | Удобство | Рекомендации |
|-------|------------|---------|-------------|
| SSH-ключи | Высокая | Высокая | Рекомендуется для постоянной работы |
| Access Token | Средняя | Высокая | Рекомендуется для CI/CD и временного доступа |
| Пароль | Низкая | Средняя | Не рекомендуется |
