# Создание и настройка репозитория

## Создание нового репозитория на gitlab.mai.ru

### Шаг 1: Создание репозитория через веб-интерфейс

1. Перейдите на [gitlab.mai.ru](https://gitlab.mai.ru)
2. Нажмите кнопку "New Project" (Новый проект)
3. Выберите один из вариантов:
   - **Create blank project** — создать пустой проект
   - **Import project** — импортировать проект из другого источника
   - **Create from template** — создать проект из шаблона

### Шаг 2: Настройка параметров проекта

При создании нового проекта необходимо указать:

- **Project name** — имя проекта (например, "lab-work-git-Фамилия")
- **Project URL** — URL проекта (автоматически генерируется на основе имени)
- **Visibility level** — уровень видимости:
  - **Private** — только вы и добавленные участники
  - **Internal** — только пользователи gitlab.mai.ru
  - **Public** — доступен всем (не рекомендуется для лабораторных работ)

Рекомендации:
- Не инициализируйте репозиторий с README, если проект уже существует локально
- Не добавляйте .gitignore и LICENSE на этом этапе, если они уже есть в проекте

### Шаг 3: Получение URL репозитория

После создания проекта GitLab покажет URL репозитория. Для SSH-подключения он будет выглядеть так:

```
git@gitlab.mai.ru:username/project-name.git
```

## Инициализация локального репозитория

Если у вас уже есть локальный проект, который нужно добавить в Git:

### Шаг 1: Переход в директорию проекта

```bash
cd path/to/your/project
```

### Шаг 2: Инициализация Git-репозитория

```bash
git init
```

### Шаг 3: Добавление файлов в индекс

Добавьте все файлы проекта в индекс:

```bash
git add .
```

Или добавьте конкретные файлы:

```bash
git add file1.txt file2.js
```

### Шаг 4: Создание первого коммита

```bash
git commit -m "Initial commit: добавлена базовая структура проекта"
```

## Привязка к удаленному репозиторию

### Шаг 1: Добавление удаленного репозитория

```bash
git remote add origin git@gitlab.mai.ru:username/project-name.git
```

Проверьте, что удаленный репозиторий добавлен:

```bash
git remote -v
```

### Шаг 2: Отправка изменений на сервер

Для первой отправки используйте флаг `-u`, чтобы установить связь между локальной и удаленной веткой:

```bash
git push -u origin main
```

Для последующих отправок достаточно:

```bash
git push
```

## Клонирование существующего репозитория

Если репозиторий уже существует на сервере, и вы хотите получить его локальную копию:

```bash
git clone git@gitlab.mai.ru:username/project-name.git
cd project-name
```

## Работа с .gitignore

Файл `.gitignore` указывает Git, какие файлы и директории игнорировать. Создайте его в корне проекта:

```bash
touch .gitignore
```

Пример содержимого `.gitignore` для различных языков:

### Для Python:

```
__pycache__/
*.py[cod]
*$py.class
*.so
.Python
env/
venv/
.venv/
.env
```

### Для Node.js:

```
node_modules/
npm-debug.log*
yarn-debug.log*
yarn-error.log*
.env
dist/
build/
```

### Для Java:

```
*.class
*.jar
*.war
*.ear
target/
.gradle/
build/
```

## Просмотр состояния репозитория

Для проверки состояния репозитория используйте:

```bash
git status
```

Эта команда показывает:
- Измененные файлы
- Файлы, добавленные в индекс
- Файлы, не отслеживаемые Git
