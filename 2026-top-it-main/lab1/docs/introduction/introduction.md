# Введение в Git и GitLab

## Что такое система контроля версий?

Система контроля версий (Version Control System, VCS) — это инструмент, который помогает отслеживать изменения в файлах (обычно в исходном коде) и координировать работу нескольких людей над одним и тем же проектом. Она сохраняет историю изменений, позволяя возвращаться к предыдущим версиям и отслеживать, кто и когда внес изменения.

## Что такое Git?

Git — это распределенная система контроля версий, созданная Линусом Торвальдсом в 2005 году для разработки ядра Linux. Основные особенности Git:

- **Распределенность**: Каждый разработчик имеет полную копию истории проекта локально
- **Скорость**: Быстрые операции с локальным репозиторием
- **Целостность данных**: Использует хэши SHA-1 для проверки целостности
- **Поддержка нелинейной разработки**: Возможность работы с множеством веток

### Основные понятия Git

- **Репозиторий** — хранилище вашего проекта с полной историей изменений
- **Коммит** — снимок состояния проекта в определенный момент времени
- **Ветка** — указатель на определенный коммит, позволяющий вести параллельную разработку
- **Слияние** — процесс объединения изменений из разных веток
- **Клонирование** — создание локальной копии удаленного репозитория
- **Push/Pull** — отправка изменений на удаленный репозиторий и получение их

## Что такое GitLab?

GitLab — это веб-платформа для управления репозиториями Git с открытым исходным кодом. Она предоставляет графический интерфейс для работы с Git и дополнительные возможности для совместной разработки:

- **Управление репозиториями** — создание, настройка и администрирование Git-репозиториев
- **Система Merge Request** — механизм для код-ревью и обсуждения изменений
- **CI/CD** — непрерывная интеграция и доставка
- **Issue Tracking** — система отслеживания задач и багов
- **Wiki** — вики-страницы для документации проекта
- **Container Registry** — хранение Docker-образов

## GitLab MAI (gitlab.mai.ru)

GitLab MAI — это внутренний сервер Московского авиационного института, предназначенный для образовательных и исследовательских целей студентов и преподавателей университета. Он предоставляет все возможности стандартного GitLab с дополнительными настройками безопасности и интеграцией с инфраструктурой университета.

### Преимущества использования gitlab.mai.ru

- **Безопасность**: Защищенное внутреннее окружение университета
- **SSH-подключение**: Безопасный доступ к репозиториям
- **Интеграция с учебным процессом**: Связь с курсами и лабораторными работами
- **Поддержка**: Техническая поддержка со стороны университета
- **Ресурсы**: Доступ к вычислительным ресурсам для CI/CD
