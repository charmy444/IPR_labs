# Логи в наблюдаемости

## Зачем нужны логи

**Логи** — это последовательность записей о событиях в приложении или инфраструктуре. В отличие от **метрик** (агрегаты) и **трейсов** (дерево одного запроса), лог обычно несёт **текстовое сообщение** и набор полей, достаточных для расследования конкретного случая.

Типичные задачи:

- понять **почему** отклонили запрос (валидация, авторизация, бизнес-правило);
- получить **stack trace** и параметры при необработанной ошибке;
- восстановить **порядок событий** на одном узле (стартап, миграции, фоновые задачи).

Ограничения: при большом трафике логи становятся **дорогими** в объёме; без **структуры** и **корреляции** с trace id их сложно сопоставлять между сервисами.

Полезные внешние материалы:

- [The Twelve-Factor App — XI. Logs](https://12factor.net/logs) — поток событий как поток stdout, агрегация отдельным слоем.
- [OpenTelemetry — Logs](https://opentelemetry.io/docs/concepts/signals/logs/) — как логи вписываются в модель OpenTelemetry и связь с трейсами.

## Структурированные логи

**Структурированный лог** (JSON, key-value) упрощает поиск и фильтрацию в системах вроде **Grafana Loki**, **Elasticsearch**, облачных продуктов. Поля вроде `level`, `service`, `trace_id`, `user_id` (осторожно с PII) позволяют строить запросы без полнотекстового хаоса.

Рекомендации по полям для корреляции:

- **`trace_id`** (и при необходимости `span_id`) — переход из лога в трейс в Grafana (Loki ↔ Tempo).
- **`request_id`** — если вы ещё не внедрили W3C Trace Context на всех путях.

## Связка лог ↔ трейс ↔ метрика

Идеальная картина для инцидента:

1. **Алерт по метрике** (рост 5xx или p99).
2. Открыть **трейс** медленного или ошибочного запроса.
3. По **trace id** найти **логи** всех задействованных подов.

В лабораторной №7 обязательный минимум — метрики и трейсы; логи и Loki остаются **рекомендуемым расширением** (см. [лаб. 7 README](../README.md)).

Ссылки:

- [Grafana — Correlate traces and logs](https://grafana.com/docs/grafana/latest/explore/correlations/) (общая идея корреляций в Explore).
- [Grafana Loki](https://grafana.com/docs/loki/latest/) — хранилище логов, часто ставят в связке с Tempo и Prometheus.

## Антипаттерны

- Логировать **секреты** и токены.
- Писать **гигабайты** отладочных логов на каждый запрос в проде без семплирования.
- Использовать только текст без уровня важности — затрудняет фильтрацию и алертинг по логам.
