# Scope Sprint 2: MVP Demo Boundaries

## Цель документа
Зафиксировать, что показывается на демо ко 2-му спринту (MVP), а что идет в post-MVP.

## Основные продуктовые ветки
1. Generate
2. Anonymize
3. Similar

## Generate (входит в MVP)
### Что пользователь делает
1. Выбирает одну или несколько таблиц для генерации.
2. Задает количество строк.
3. Получает CSV-файл(ы) на скачивание.

### Таблицы генерации в MVP и их колонки
1. `users.csv`
- `user_id`
- `full_name`
- `email`
- `phone`
- `city`
- `address`
- `birth_date`
- `registration_date`

2. `orders.csv`
- `order_id`
- `user_id`
- `product_id`
- `amount`
- `order_date`
- `currency`

3. `payments.csv`
- `payment_id`
- `order_id`
- `user_id`
- `status`

4. `products.csv`
- `product_id`
- `name`
- `price`

5. `support_tickets.csv`
- `ticket_id`
- `user_id`
- `created_at`
- `message`
- `status`
- `priority`
- `operator_name`

## Anonymize (входит в MVP)
### Что пользователь делает
1. Загружает любой CSV.
2. Получает анализ колонок.
3. Вручную выбирает обработку для каждой колонки.
4. Скачивает анонимизированный CSV.

### Что входит в MVP
- Работа с произвольным CSV, а не только с заранее заданными шаблонами.
- Пошаговый сценарий "загрузка -> анализ -> выбор по колонкам -> скачивание".
- Для каждой колонки значение по умолчанию: `keep`.
- После анализа файла система показывает тип колонки, превью и сразу помечает методы, которые к ней не подходят.
- Стабильный результат для демо на типовых датасетах.

## Similar (post-MVP фича, не входит в демо 2-го спринта)
### Что пользователь делает
1. Загружает CSV.
2. Система анализирует структуру и базовые статистики.
3. Получает похожий synthetic CSV на скачивание.

### Что включает фича
- Анализ схемы (колонки/типы) и простых статистик (распределения, диапазоны, категории).
- Генерация "похожего" набора данных с сохранением структуры колонок.
- Экспорт результата в CSV.

## Разделение по спринтам
### Ко 2 спринту (MVP demo)
- Две рабочие ветки: `Generate`, `Anonymize`.
- Демо-сценарии:
  - Generate: выбор таблиц + rows -> скачивание CSV.
  - Anonymize: загрузка CSV -> анализ колонок -> ручная настройка (default `keep`) -> скачивание.

### Post-MVP (спринты 3-4)
- Реализация подсказок правил анонимизации (`suggest_pii` / `suggest_rules`) с `suggested_method`, `suggestion_confidence` и `pii_hint`.
- Реализация фичи `Similar`:
  - загрузка CSV,
  - анализ структуры и статистик,
  - генерация похожего synthetic CSV.
- Улучшение качества synthetic-генерации и сохранения зависимостей.
- Повышение надежности, покрытие edge-cases, UX-доработки.

### Optional (по возможности)
- Расширенные метрики качества синтетики и приватности.
- Дополнительные форматы файлов и интеграции.
- Расширенный набор готовых доменных шаблонов.
